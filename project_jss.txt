=====


jss
  -> json with syntax sugar
     (inspired from json5 and yaml)

The all point of this language is to generate JSON
from a language that allows one to write
JSON-like structured data in the most concise
and convenient way.

This kind of files should be written by hand,
intentionally, rather than following
"good practices/conventions", and should
absolutely not be generated programatically.

---

The syntax borrows ideas from json5:
  -> strings can be barewords (meaning they
     don't need to be quoted) as long as
     it doesn't conflict with the language
  -> single line comment and side comment
  -> inline comment and multiline comment
  -> numbers may be of various forms
     (hexadecimal, octal, binary, fraction,
      power, periodic decimals, ..)
      
The syntax borrows ideas from yaml:
  -> multiline structure such as lists
     and maps are written using YAML-like
     syntax
  -> inline json as terminals

Basically start from the leafs with inline JSON,
and then build with yaml and nested yaml.

- `,` + space => list/map separator
- `:` + space => key-val separator

Metacharacters (not allowed in symbols):
  -> {}
  -> []
  -> (space)
  -> (newline)
